# 法术大全 · 需求分解

与职业库、专长表、武技库平行，供**法术大全页**查阅与**角色卡**调用。

---

## 一、数据层（可先做）

**文件**：`src/data/spellDatabase.js`

**单条法术结构建议**：

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | string | 唯一标识，如 `fire_ball`、`cure_wounds` |
| `name` | string | 中文名 |
| `level` | number | 环阶：0=戏法，1–9=一环至九环 |
| `school` | string? | 学派：防护/咒法/预言/惑控/塑能/幻术/死灵/变化 等 |
| `source` | string[]? | 法表来源：`['法师','牧师']` 等，供“按职业筛选”与角色卡匹配 |
| `ritual` | boolean? | 是否可仪式施法 |
| `castingTime` | string? | 施法时间 |
| `range` | string? | 施法距离 |
| `components` | string? |  V/S/M 等 |
| `duration` | string? | 持续 |
| `description` | string | 效果描述（可较长） |
| `sourceBook` | string? | 出处：核心/繁星/模组 等 |

**导出与辅助函数**：

- `SPELLS`：法术数组（或按环阶/学派分组，看录入与展示习惯）。
- `getSpellById(id)`：角色卡用 id 查单条。
- `getSpellsByLevel(level)`：按环阶筛选。
- `getSpellsByClass(className)`：按职业法表筛选（`source` 含该职业即入选）。
- `getSpellsBySchool(school)`、`searchSpells(nameOrKeyword)`：法术大全页筛选/搜索用。

**录入方式**：先以「按环阶」为主（戏法 + 1–9 环），每条满足上表即可；同一法术多职业时在 `source` 里多写几个即可。

## 二、法术大全页（`/spells`）

**现状**：`Spells.jsx` 仅有标题与“开发中”占位。

**建议**：

1. **列表与筛选**  
   - 数据来自 `spellDatabase`。  
   - 筛选：环阶（0–9）、学派、职业法表（来源）、是否仪式；可选关键词搜索。  
   - 列表展示：名称、环阶、学派、来源（职业）、仪式标签；点击展开看完整描述（同专长表折叠）。

2. **分栏方式**（可选）  
   - 与专长表类似：按「环阶」分栏（戏法 / 一环 / 二环 / …），每栏下为法术列表并可折叠详情。  
   - 或先做平铺列表 + 筛选项，后续再加分栏。

3. **与房规关系**  
   - 若希望和职业库/专长表同一入口：可在房规页加「法术大全」折叠区块，内嵌同一列表或跳转 `/spells`。  
   - 若独立使用多：保持 `/spells` 独立页即可，角色卡与房规都从同一数据源读。

## 三、角色卡调用

**目标**：角色卡能展示并编辑「戏法 / 已知法术 / 准备法术」，选项来自法术大全数据。

1. **角色数据结构**  
   在现有角色对象上增加，例如：  
   - `spells.cantrips`：戏法 id 数组。  
   - `spells.known`：已知法术 id 数组（法师书记录、术士已知等）。  
   - `spells.prepared`：当前准备法术 id 数组（牧师/德鲁伊/法师等）。  
   若存在多施法来源（如兼职），可用 `spells.bySource: { 法师: { known: [], prepared: [] }, 牧师: { prepared: [] } }` 等，后续再细化。

2. **角色卡 UI**  
   - 在已有「施法等级 / 契约等级 / 施法关键属性」附近或下方增加「法术」区块。  
   - 当 `getSpellcastingLevel(char) > 0` 或 `getPactLevel(char) > 0` 时显示。  
   - 戏法：多选或列表，从 `getSpellsByLevel(0)` 且 `getSpellsByClass(char.class)` 的交集中选（或按多职业合并法表）。  
   - 已知/准备：同上，用 `getSpellsByLevel(1)`…`getSpellsByLevel(9)` 与职业法表过滤，选择结果写入 `char.spells`。  
   - 展示时用 `getSpellById(id)` 显示名称、环阶、学派；点击可展开描述或跳转法术大全。

3. **法术位**  
   - 若已有法术位表（按施法等级查表），保持不变；角色卡只负责「有哪些法术」与「准备了哪些」，消耗仍按规则在战斗/记录中处理。

---

## 四、实施顺序建议

| 步骤 | 内容 | 说明 |
|------|------|------|
| 1 | 建 `spellDatabase.js` | 定义 `SPELLS` 与 getSpellById / getSpellsByLevel / getSpellsByClass，先空数组或少量示例 |
| 2 | 法术大全页接数据 | Spells.jsx 读 SPELLS，按环阶/学派/职业筛选 + 搜索，列表 + 折叠详情 |
| 3 | 开始录入 | 按环阶分批录入，补全 name/level/school/source/description 等 |
| 4 | 角色数据结构 | 在 character 上增加 `spells` 字段，characterStore 默认值 |
| 5 | 角色卡法术区块 | 施法者显示戏法/已知/准备列表，选择器从 spellDatabase 按职业+环阶过滤 |
| 6 | 可选 | 房规页增加「法术大全」折叠入口；或法术大全页增加「按来源（核心/繁星）」分栏 |

这样分解后，你可以先完成 1–3（数据层 + 大全页 + 录入），角色卡调用在 4–5 再做，两者都依赖同一套 `spellDatabase`，便于后续扩展（如升环、专注、材料等字段）。
