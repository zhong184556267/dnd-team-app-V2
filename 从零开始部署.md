# 从零开始部署 D&D 团队小助手

这是一份**完全从零**的部署教程，按顺序做就能得到一个可以分享给队友的网址。  
你不需要会写代码，只要会复制粘贴、点网页即可。

---

## 你需要准备的东西

| 项目 | 说明 |
|------|------|
| **电脑** | Windows / Mac 都可以 |
| **浏览器** | Chrome、Edge 等 |
| **Node.js** | 用来在本地运行和打包项目，下面会教安装 |
| **GitHub 账号** | 免费注册，用来放代码，Vercel 需要它 |
| **（可选）Supabase 账号** | 若需要「10 人共享角色 + 团队仓库」，再注册 |

---

## 第 0 步：安装 Node.js

1. 打开：**https://nodejs.org**
2. 下载页面推荐的那个 **LTS 版本**（左边绿色按钮），双击安装。
3. 安装时一路「下一步」即可，默认选项不用改。
4. 装好后：
   - 按 **Win + R**，输入 **cmd** 回车，打开黑色命令行窗口；
   - 输入：`node -v` 回车，若出现类似 `v20.x.x` 就说明装好了。

---

## 第 1 步：拿到项目代码

- **如果项目已经在你这台电脑上**（例如你有一个 `dnd-team-app` 文件夹）：  
  用资源管理器打开这个文件夹，记住路径，例如：`G:\dnd-team-app`。

- **如果还没有代码**：  
  问给你项目的人要压缩包或 Git 地址，解压到一个文件夹（如 `G:\dnd-team-app`）。

---

## 第 2 步：在本地先跑起来（确认没问题）

1. 打开**命令提示符**（Win + R → 输入 `cmd` → 回车）。
2. 进入项目文件夹（把路径改成你的实际路径）：
   ```bash
   cd /d G:\dnd-team-app
   ```
3. 安装依赖（第一次需要，可能要等一两分钟）：
   ```bash
   npm install
   ```
4. 启动本地网站：
   ```bash
   npm run dev
   ```
5. 终端里会显示一行类似：`Local: http://localhost:5173/`  
   用浏览器打开这个地址，能看到登录页就说明项目没问题。
6. 关掉网站：在终端里按 **Ctrl + C**。

> 说明：这一步只是在你电脑上能访问，队友还访问不到。下面要把网站「部署到网上」。

---

## 第 3 步：把代码放到 GitHub（Vercel 需要）

1. 若没有 GitHub 账号，先打开 **https://github.com** 注册并登录。
2. 登录后，点击右上角 **「+」** → **「New repository」**（新建仓库）。
3. 填写：
   - **Repository name**：填 `dnd-team-app`（或任意英文名）。
   - 选 **Public**。
   - 不要勾选 "Add a README file"（我们本地已有代码）。
4. 点 **「Create repository」**。
5. 在**你的项目文件夹**里打开命令行（同上，`cd /d G:\dnd-team-app`），依次执行（**把 `你的GitHub用户名` 换成你真名**）：

   ```bash
   git init
   git add .
   git commit -m "初始提交"
   git branch -M main
   git remote add origin https://github.com/你的GitHub用户名/dnd-team-app.git
   git push -u origin main
   ```

   若提示要登录 GitHub，按提示在浏览器里用账号授权即可。

6. 刷新你的 GitHub 仓库页面，能看到一堆代码文件就说明推送成功。

---

## 第 4 步：用 Vercel 部署成网址（免费）

1. 打开 **https://vercel.com**，用 **GitHub** 登录（和 GitHub 绑定一下）。
2. 登录后点 **「Add New...」** → **「Project」**。
3. 在列表里找到你刚推送的 **dnd-team-app**，右侧点 **「Import」**。
4. **Configure Project** 页面：
   - **Build Command**：保持默认（一般是 `npm run build`）。
   - **Output Directory**：保持默认 `dist`。
   - 若你**暂时不配置 Supabase**（先只要一个能打开的网址）：
     - 下面 **Environment Variables** 可以先不填，直接点 **「Deploy」**。
   - 若你**已经按第 5 步配好了 Supabase**：
     - 在 **Environment Variables** 里添加两条：
       - **Name**：`VITE_SUPABASE_URL`，**Value**：你的 Supabase Project URL。
       - **Name**：`VITE_SUPABASE_ANON_KEY`，**Value**：你的 Supabase anon public 密钥。
     - 再点 **「Deploy」**。
5. 等一两分钟，出现 **Congratulations** 和一行网址（如 `https://dnd-team-app-xxx.vercel.app`），就部署成功了。

之后你可以把**这个网址**发给队友，大家用浏览器打开，输入自己的名字就能用。

> **数据说明**：  
> - 若第 4 步**没填** Supabase 环境变量：角色和仓库数据只存在**各自浏览器**里，换设备或清空数据会丢，且无法多人共享同一套仓库。  
> - 若**填了** Supabase（并完成下面第 5 步）：角色按人存云端，团队仓库按模组共享，可多人一起用、物品交换。

---

## 第 5 步（可选）：配置 Supabase——实现 10 人共享角色 + 团队仓库

只有在需要「每人保存自己的角色 + 团队仓库统一管理、物品交换」时才做这一步。

### 5.1 注册并创建 Supabase 项目

1. 打开 **https://supabase.com**，点 **「Sign in」**，用 **GitHub** 或 **Google** 登录。
2. 进入后点 **「New project」**。
3. **Name** 填 `dnd-team-app`（或任意），**Database Password** 自己设一个并记住，**Region** 选 **Singapore** 或 **Tokyo**。
4. 点 **「Create new project」**，等 1～2 分钟。

### 5.2 拿到两个密钥

1. 左侧点 **「Settings」** → **「API」**。
2. 记下：
   - **Project URL**（一整行，如 `https://xxxxx.supabase.co`）→ 用作 `VITE_SUPABASE_URL`。
   - **Project API keys** 里 **anon public** 那一行的密钥（点复制）→ 用作 `VITE_SUPABASE_ANON_KEY`。

### 5.3 在 Supabase 里建表

1. 左侧点 **「SQL Editor」** → **「New query」**。
2. 打开你电脑里项目中的 **`supabase-schema-v2.sql`** 文件（在 `dnd-team-app` 文件夹里），**Ctrl+A 全选**，**Ctrl+C 复制**。
3. 回到 Supabase 网页，把复制的内容**粘贴**到左侧输入框（不要带 \`\`\` 这种标记）。
4. 点右下角 **「Run」**。
5. 显示 **「Success. No rows returned」** 即成功。

### 5.4 在本地填密钥（方便本地调试）

1. 在项目文件夹里找到 **`.env.example`**，**复制一份**，重命名为 **`.env`**。
2. 用记事本打开 `.env`，改成（替换成你 5.2 记下的两个值）：
   ```
   VITE_SUPABASE_URL=https://你的项目id.supabase.co
   VITE_SUPABASE_ANON_KEY=你复制的anon密钥
   ```
3. 保存。

### 5.5 在 Vercel 里填环境变量并重新部署

1. 打开 **https://vercel.com**，进入你的 **dnd-team-app** 项目。
2. 点顶部的 **「Settings」** → 左侧 **「Environment Variables」**。
3. 添加两条：
   - **Key**：`VITE_SUPABASE_URL`，**Value**：你的 Project URL，保存。
   - **Key**：`VITE_SUPABASE_ANON_KEY`，**Value**：你的 anon 密钥，保存。
4. 点顶部 **「Deployments」**，在最新一次部署右侧点 **「⋯」** → **「Redeploy」**，确认再部署一次。

完成后，用 Vercel 的网址访问，角色和团队仓库就会存到 Supabase，多人共享。

## 常见问题

**Q：没有 Git，命令行里 `git` 报错？**  
A：先安装 Git：https://git-scm.com/download/win ，安装后再执行第 3 步的命令。

**Q：Vercel 部署后打开网页是白的？**  
A：若你配了 Supabase，检查 Vercel 的 Environment Variables 里 `VITE_SUPABASE_URL` 和 `VITE_SUPABASE_ANON_KEY` 是否填对、没有多余空格。

**Q：不配 Supabase 可以吗？**  
A：可以。不填 Supabase 相关环境变量，网站照样能打开，只是数据只在各自浏览器里，不能多人共享同一套角色和仓库。

**Q：以后改代码怎么更新网站？**  
A：在项目文件夹里改完代码后，用 `git add .`、`git commit -m "更新说明"`、`git push` 推到 GitHub，Vercel 会自动重新部署。
