# 团队使用说明（约 10 人）

功能已就绪，可以让大家一起用了。按下面步骤部署并分享链接即可。

> **第一次部署、不太会操作？** 请先看 **《从零开始部署.md》**，从安装 Node.js 到拿到网址、可选配置 Supabase，都有一步步说明。

---

## 一、部署（拿到一个可访问的链接）

任选一种方式，把项目变成「一个网址」。

### 方式 A：Vercel（推荐，免费）

1. 把本仓库推到 **GitHub**（若还没有）。
2. 打开 [vercel.com](https://vercel.com)，用 GitHub 登录。
3. 点击 **Add New → Project**，导入你的 `dnd-team-app` 仓库。
4. **Build and Output Settings** 保持默认即可（Vite 会自动识别）：
   - Build Command: `npm run build`
   - Output Directory: `dist`
5. 点击 **Deploy**，等一两分钟。
6. 部署完成后会得到一个地址，例如：`https://dnd-team-app-xxx.vercel.app`。

之后每次往 GitHub 推送代码，Vercel 会自动重新部署。

### 方式 B：其他静态托管

在项目根目录执行：

```bash
npm run build
```

把生成的 **`dist`** 文件夹里的全部内容上传到你使用的静态托管（如 Netlify、Cloudflare Pages、自己的服务器等），并确保「未匹配路径都指向 `index.html`」（SPA 回退）。

## 二、分享给小伙伴（约 10 人）

1. 把部署好的链接（如 `https://xxx.vercel.app`）发给大家。
2. 告诉大家：
   - 用**手机或电脑浏览器**打开这个链接。
   - 第一次会看到**登录页**，在「请输入角色名」里输入**自己的名字或昵称**（例如：小明、战士老王），点「进入繁星世界」。
   - 之后角色、背包、仓库等数据都会按「当前登录的名字」区分，**数据存在各自浏览器里**，互不干扰。

每人用一个自己记得住的名字即可，无需注册账号。

## 三、使用上的说明（可转发给队友）

- **DM**：若你希望某人为管理员，在 `src/lib/auth.js` 里把 `ADMIN_NAME` 改成该名字（如 `'AdminBEAR'`），重新部署后，用这个名字登录的人即为管理员。
- **角色与仓库**：每人登录后看到的是「自己的角色列表」和「当前模组下的团队仓库」；仓库、货币等是**按模组共享**的（同一链接、同一模组下大家看到同一套仓库）。
- **数据在哪**：
  - **未配置 Supabase**：角色和仓库数据只存在**各自设备的浏览器**里，换设备或清空数据会丢失，且**无法实现多人共享同一套团队仓库与角色持久化**。
  - **配置 Supabase 后**：角色按「当前登录名」保存到云端，每人只能编辑自己的角色；团队仓库按模组存到云端，**多人共享同一套仓库、可进行角色与仓库之间的物品交换与统一管理**。需要跨设备或 10 人真正共享时，请务必按下面步骤配置 Supabase。
- **10 人共享角色 + 团队仓库（必配 Supabase）**：若需要「每人创建并保存自己的角色」且「团队仓库统一管理、不同角色之间物品交换」，请在项目中配置 Supabase，并执行 **`supabase-schema-v2.sql`** 创建/更新表结构（角色表 `characters`、仓库表 `warehouse`）。详见 **《Supabase部署教程-零基础.md》**。

## 四、小结

| 步骤       | 你要做的 |
|------------|----------|
| 部署       | 用 Vercel 或其它方式把项目部署成一个网址 |
| 分享       | 把该网址发给约 10 位小伙伴 |
| 大家使用   | 每人打开链接 → 输入自己的名字 → 开始用 |
| 多人共享   | **若要每人保存自己的角色 + 团队仓库统一管理/物品交换**：必须配置 Supabase 并执行 `supabase-schema-v2.sql` |

这样约 10 人就可以同时使用；需要「角色云端保存 + 团队仓库共享」时，配置 Supabase 即可。
