# 繁星 D&D 小助手 · 最终执行规划书 (Starlight D&D Assistant v2.0)

---

## 1. 核心架构

- **用户角色**
  - **DM（管理员）**：输入名字 `AdminBEAR`（可配置）视为 DM，拥有所有权限。
  - **玩家**：输入其他名字，拥有受限权限。
- **数据存储**：JSONBin.io（云端 JSON），单团单 Bin。
- **部署方式**：Vercel 静态托管。
- **终端适配**：Mobile First，大按钮、卡片式布局、底部导航栏。

---

## 2. 技术约定（已定项）

- **数据模型**：单团单 Bin；一个登录用户对应多张角色卡，角色列表按「当前登录名」过滤。开新团再考虑多 Bin。
- **登录**：记住当前用户（localStorage），下次打开自动恢复。
- **头像**：先用 Base64 存进角色数据，不接图床；后续可改为图床 URL。
- **首页**：不做「当前在线角色数」，可展示「本团角色数」等静态统计或删掉。

---

## 3. 核心计算引擎（自动化逻辑）

所有数据变动需触发以下自动计算：

- **属性调整值**：`Math.floor((属性值 - 10) / 2)`。
- **经验与等级**：输入单次获得经验 → 累加总经验 → 查表自动换算当前等级。
- **AC（护甲等级）**：10 + 敏捷调整值 + 护甲基础值 + 盾牌值 + 其他装备加成 + Buff。
- **技能/豁免检定**：d20 + 属性调整值 + 熟练加值（按等级查表）+ 专精加成。
- **HP**：当前 HP = 上限 + 临时 HP - 累计伤害；支持「一键扣除 X 点」。
- **多货币**：基础单位 GP；其他币种配置与 GP 汇率；显示「总资产 (GP)」并保留各币种数量。

## 4. 页面与功能（移动端优化）

### 页面 0：登录页 (Login)

- **UI**：居中 Logo，大输入框「请输入角色名」，大按钮「进入繁星世界」。
- **逻辑**：输入 `AdminBEAR` → isAdmin = true；其他 → isAdmin = false，currentUser = 输入名。记住到 localStorage。

### 全局底部导航栏 (Bottom Tab Bar)

- 首页 (Dashboard)
- 我的角色 (Characters)
- 团队仓库 (Warehouse)
- 法术大全 (Spells)
- 更多 (More)：繁星特色、数据维护(DM)、登出

### 页面 1：首页 (Dashboard)

- 欢迎语。
- 快捷工具：万能骰子（d4–d100，优势/劣势）、快速记账、公告栏、资产概览（团队总资金折算 GP）。

### 页面 2：角色卡系统

- **列表页**：卡片式，头像/名字/等级/职业/HP 条；玩家只看自己或只读队友，DM 看全部。
- **详情/编辑页**（Tab）：基础信息、职业与经验、属性与技能、战斗状态（HP/AC/Buff）、装备与背包、日志。

### 页面 3：团队仓库

- 资金面板（多币种输入，底部总计 GP）。
- 团队物品 / 个人物品预览；新增、数量调整、分配、制作；记账本（时间轴 + 「记一笔」）。

### 页面 4：法术大全

- 只读 + 搜索（名称、环阶、学派）；卡片列表；详情展开。DM 在「更多」中维护。

### 页面 5：繁星特色（房规/模组）

- 富文本展示；全员只读，DM 可编辑。

### 隐藏数据库（支撑计算）

- 物品百科、职业表、专长表；前端对玩家隐藏列表，仅在选择器中调用；DM 在「更多」→ 数据维护中编辑。
